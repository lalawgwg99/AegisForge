# Problem Map: 4 ecosystems, shared pain points

## Shared pain points
1. Session amnesia / repeated mistakes
2. Memory grows without quality control
3. Weak safety boundaries for autonomous operations
4. Hard to compare improvements objectively
5. Fragmented channel and runtime experiences

## Strengths to borrow

### Claude Code
- Strong coding workflow ergonomics
- Reliable iteration flow in code tasks

### GPT Codex
- High-velocity code synthesis and task decomposition
- Strong fit for scaffold-heavy engineering work

### OpenClaw
- Excellent multi-channel gateway model
- Real-world personal assistant operations (DMs/channels/devices)

### Hermes Agent
- Tool-first orchestration
- Skills + memory + cron + delegation patterns

## Fusion strategy
- Keep OpenClaw-style multi-channel control plane
- Add Hermes-style tool/skill/memory primitives
- Keep Claude/Codex-style coding UX and execution speed
- Add a strict reliability layer (learning + forgetting + policy gates)

## Non-goals
- Not replacing each product
- Not cloning UI/branding
- Not forcing single-model dependency
